Titles (Headers)

For primary headline text

Titles are defined by classes independently of the HTML tag they may be applied to. As such, any tag can look like a header. By default, the logical numbered Title tag styles are applied to the corresponding numbered H-tag, so if you need an H2 and it looks like a .t2, you can ommit the class name.

Any of these can be made bold (which may not be the 'bold' font) by including a <strong> tag inside. This will cause a 'light' font to be 'regular', a 'regular' font to be 'medium' and a 'medium' font to be 'bold'. This will be annotated in the PSD's by -b, for instance, .t3-b.

Labels (Subheaders)

x

Labels are defined solely by classes, and can define the style of any H-tag, P, etc.

Any of these can be made bold (which may not be the 'bold' font) by including a <strong> tag inside. This will cause a 'light' font to be 'regular', a 'regular' font to be 'medium' and a 'medium' font to be 'bold'.

All labels also have a "light" version, which is implemented by adding "-light" to the class name, for instance, .l2-light.

Buttons

Default buttons

Button styles can be applied to anything with the .btn class applied. However, typically you'll want to apply these to only <a>, <button> or <input> elements.

In most cases <a> buttons will be most appropriate.

Standard Button
Get Sony Support
  <a class="btn">...</a>
'Special' Button – Generally used for store links
Where to Buy
  <a class="btn btn-special">...</a>
'Alt' Button
Alt Button
  <a class="btn btn-alt">...</a>

Button sizes

Fancy larger or smaller buttons? Add .btn-large, .btn-small, or .btn-mini for additional sizes.

Large buttonDefault buttonSmall buttonMini button
  <a class="btn btn-large">Large button</a> 
  <a class="btn">Default button</a> 
  <a class="btn btn-small">Small button</a> 
  <a class="btn btn-mini">Mini button</a>

Block level buttons Create block level buttons—those that span the full width of a parent— by adding.btn-block.

Block level buttonBlock level special button
  <a class="btn btn-block">Block level button</a> 
  <a class="btn btn-block btn-special">Block level special button</a>

Disabled state

Make buttons look unclickable by fading them back 50%.

Anchor element

Add the .disabled class to <a> buttons.

Link Special link

  <a href="#" class="btn disabled">Link</a> 
  <a href="#" class="btn btn-special disabled">Special link</a>

Heads up! We use .disabled as a utility class here, similar to the common .active class, so no prefix is required. Also, this class is only for aesthetic; you must use custom JavaScript to disable links here.

Button element

Add the disabled attribute to <button> buttons.

 

  <button type="button" class="btn" disabled>Button</button> 
  <button type="button" class="btn btn-special disabled" disabled="disabled">Special button</button>

One class, multiple tags

Use the .btn class on an <a>, <button>, or <input> element.

Link   
  <a class="btn" href=""> Link</a> 
  <button class="btn" type="submit"> Button</button> 
  <input class="btn" type="button" value="Input"> 
  <input class="btn" type="submit" value="Submit">

As a best practice, try to match the element for your context to ensure matching cross-browser rendering. Usually you'll make an a anchor link. If you have an input, use an <input type="submit"> for your button. If you have a button, use <button type="button">.

Sections with a light blue background have not been styled or updated yet. If you update them, remove the class "NotUpdatedYet" from the parent container of the section, and from the "bs-docs-example" container as well.

Inputs

Most common form control, text-based input fields. Includes support for all HTML5 types: text, password, datetime, datetime-local, date, month, time, week, number, email, url, search, tel, and color.

Requires the use of a specifiedtypeat all times.

<input type="text" placeholder="City or Postal Code">

Textarea

Form control which supports multiple lines of text. Changerowsattribute as necessary.

<textarea rows="3"> </textarea>

Control sizing

Use relative sizing classes like .input-large or match your inputs to the grid column sizes using .span* classes.

Relative sizing
<input class="input-mini" type="text" placeholder=".input-mini"> 
<input class="input-small" type="text" placeholder=".input-small"> 
<input class="input-medium" type="text" placeholder=".input-medium"> 
<input class="input-large" type="text" placeholder=".input-large"> 
<input class="input-xlarge" type="text" placeholder=".input-xlarge"> 
<input class="input-xxlarge" type="text" placeholder=".input-xxlarge">

Heads up!In future versions, we'll be altering the use of these relative input classes to match our button sizes. For example, .input-large will increase the padding and font-size of an input.

Grid sizing

Use .span1 to .span12 for inputs that match the same sizes of the grid columns.

<input class="span1" type="text" placeholder=".span1"> 
<input class="span2" type="text" placeholder=".span2"> 
<input class="span3" type="text" placeholder=".span3">
